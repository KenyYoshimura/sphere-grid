## 第1章：設計思想と目的

### 1.1 従来システムからの脱却

従来のRPGにおける成長システムの問題点：
- **レベルアップ制の受動性**：経験値が溜まると自動的にステータスが上昇し、プレイヤーの介入余地がない
- **成長の不透明性**：次のレベルで何が得られるか分からない
- **キャラクター個性の埋没**：全員が同じ成長曲線を辿る

スフィア盤が解決したこと：
- **能動的な成長選択**：プレイヤーが「何を」「いつ」「どの順番で」成長させるか決定
- **成長の完全可視化**：将来獲得できる全能力が最初から見える
- **キャラクター分岐の視覚化**：各キャラの専門領域と他キャラとの関係性が一目で理解できる

### 1.2 コアデザインコンセプト

**「成長を旅にする」**

スフィア盤は単なるステータス管理画面ではなく、キャラクターの成長を「空間的な移動」として表現することで、以下の体験を生み出しています：

1. **目的地の存在** — 欲しいアビリティが「あそこにある」と視認できる
2. **道程の実感** — そこに辿り着くまでの距離と必要リソースが分かる
3. **寄り道の誘惑** — ルート上に魅力的なノードがあり、戦略的判断を迫られる
4. **到達の達成感** — 物理的に「移動して獲得した」という手応え

## 第4章：プレイヤー心理設計

### 4.1 モチベーション維持メカニズム

#### 短期目標の連続設計
```
【心理サイクル】

「あのアビリティが欲しい」（欲求）
      ↓
「あと5マス進めば届く」（見通し）
      ↓
「今回の戦闘でSレベル2上がった」（進捗実感）
      ↓
「やっと習得！」（達成感）
      ↓
「次はあっちのアビリティを…」（新たな欲求）
```

#### 達成感の演出要素
| 要素 | 演出 |
|-----|------|
| ノード発動時 | 光のエフェクト + 効果音 |
| アビリティ習得時 | 専用ジングル + テキスト演出 |
| 新エリア到達時 | ズームアウトで新領域表示 |
| ロック解除時 | 鎖が砕けるエフェクト |

### 4.2 選択の自由度と責任感

#### 不可逆性の設計
- **移動**：取り消し不可（Sレベルは戻らない）
- **発動**：取り消し不可（スフィアは戻らない）
- **ロック解除**：取り消し不可

**設計意図**：
- 「考えて選ぶ」重要性を高める
- 自分の選択に対するオーナーシップ
- 「失敗したら最初からやり直し」というやり込み要素

#### セーフティネット
- 通常プレイでは「詰み」がない設計
- どのルートを選んでもクリア可能
- 効率の差はあれど、全てのノードは最終的に発動可能

### 4.3 先の見通しと計画性

#### 完全情報公開の設計
```
【見えるもの】
✓ スフィア盤全体のノード配置
✓ 各ノードの効果
✓ ロックの位置とレベル
✓ 他キャラの現在位置
✓ 他キャラのルートとの合流点

【見えないもの】
✗ 特殊スフィアで何を配置できるか（中盤以降に解放）
✗ 効率的なルート（プレイヤーが発見する）
```

**設計意図**：
- 「知っていれば有利」ではなく「誰でも計画できる」公平性
- 情報を隠さないことでストレス軽減
- 計画を立てる楽しさの提供

## 第5章：戦略的深度

### 5.1 リソース管理の多層構造

```
【管理すべきリソース】

第1層：AP（アビリティポイント）
  └─ 戦闘で獲得、Sレベルに変換される

第2層：Sレベル
  └─ 移動に消費、貯蓄も可能

第3層：スフィア（発動用）
  └─ 敵ドロップ、盗む、宝箱で入手
  └─ 種類ごとに発動できるノードが異なる

第4層：キースフィア
  └─ ボス撃破、特定条件で入手
  └─ 希少、計画的使用が必要

第5層：特殊スフィア（終盤）
  └─ 空ノード活用、最終育成用
```

### 5.2 ルート選択のジレンマ

#### 典型的な選択シナリオ
```
【シナリオ1：直進 vs 寄り道】

現在地 ──→ HP+300（3マス）──→ アルテマ（目標）
    │
    └──→ ケアルガ（2マス、別方向）

判断要素：
・HPはすぐ欲しいか？
・ケアルガは今必要か？
・Sレベルの余裕は？
・スペシャルスフィアは足りているか？
```

```
【シナリオ2：ロック突破 vs 迂回】

現在地 ──→ [Lv2ロック] ──→ ヘイスガ（近道）
    │
    └──→ 遠回りルート（ロックなし、+8マス）

判断要素：
・Lv2キースフィアは何個ある？
・他キャラで使う予定は？
・遠回りルート上に有用なノードは？
```

### 5.3 キャラクター間シナジー

#### 先行キャラ・後続キャラ戦略
```
【効率的育成パターン】

1. ティーダ（高速キャラ）がルールー領域へ侵入

      ↓
3. ルールーがティーダの軌跡を辿る
      ↓
4. 既発動ノードはスフィア消費なしで効果獲得
      ↓

```

#### フレンドスフィア活用
```
【長距離ワープ戦略】

リュック現在地（リュック領域末端）
      │
      │ フレンドスフィア使用
      ↓
アーロン現在地（アーロン領域中央）へワープ
      │
      │ アーロン領域の力ノードを発動
      ↓

#### 確認ダイアログの出現条件
```

## 第10章：スタンダード盤 vs エキスパート盤

### 10.1 比較表

| 項目 | スタンダード盤 | エキスパート盤 |
|-----|---------------|---------------|

- 2周目以降のリプレイバリュー向上

## 第11章：UI/UXの革新性まとめ

### 11.1 スフィア盤が成功した理由

1. **成長の空間化**
   - 抽象的な「レベルアップ」を具体的な「移動」に置換
   - 「どこに行くか」という直感的な選択に変換

2. **完全な情報開示**
   - 隠し要素を最小化し、計画性を重視
   - 「知らなかったから損した」を防止

3. **適切な複雑さ**
   - 基本操作は単純（移動して発動）
   - 深い戦略は任意（やり込み層向け）

4. **視覚的フィードバック**
   - 進捗が「見える」
   - 達成が「光る」

   - 他の誰とも違う成長軌跡
